План подготовки к собеседованиям на AQA Java

Собери фундамент, закрепи практикой и тренируй собес‑навыки как отдельный навык.

«Код писать могут ИИ. Но собеседование за тебя не пройдёт ИИ».
Твоя цель — уметь объяснить решения, компромиссы и ход мысли.

1. Java Collections →
  • List, Set, Map — база для обработки данных в тестах.

2. Mocks & Stubs →
  • Mockito — учимся изолировать систему от внешних сервисов.

3. API Testing

  • RestAssured — учимся тестировать бэкенд через код.

4. UI Testing

  • Selenide/Playwright — визуализация действий пользователя.
Основные темы

1) Java Core

  • OOP, SOLID, исключения, generics
  • Collections, Stream API, Optional
  • Concurrency: threads, executors, locks
  • JVM basics: memory, GC, classloading

2) Тестирование (теория)

  • Пирамида тестов, тест‑дизайн
  • Flaky tests, стабильность и изоляция
  • Что/где мокать: unit vs integration
  • Test data, determinism, idempotency

3) API Testing

  • HTTP, статусы, headers, cookies
  • RestAssured: позитив/негатив/контракт
  • Auth: session/JWT, refresh/expiry
  • Логи, retries, тесты ошибок

4) UI Testing

  • Selenide: Page Object, стабильные локаторы
  • Ожидания, синхронизация, сетевые паузы
  • Скриншоты, видео, отчёты Allure
  • Параллельный запуск, конфиги браузера

5) Spring Boot & Data

  • Controllers, validation, exception handling
  • JPA basics: entities, relations, transactions
  • Testcontainers / embedded DB (где уместно)
  • Профили окружений: dev/test

6) CI/CD & качество

  • Стабильный pipeline, отчёты, артефакты
  • Параллель, шардирование, quarantine
  • Code review, читаемость тестов
  • Метрики: duration, flaky rate
Совет: 2–3 раза в неделю делай «мини‑собес» — объясни вслух один тест/архитектурное решение за 5 минут.
