## 1. Объект тестирования

| Элемент | ID / Селектор | Назначение |
|---------|---------------|------------|
| Поле ввода JSON | `#jsonInput` | Ввод сырого JSON |
| Кнопка «Отформатировать» | `#formatBtn` | Запуск форматирования и построения дерева |
| Кнопка «Очистить» | `#clearBtn` | Очистка всех полей |
| Кнопка «Пример» | `#exampleBtn` | Загрузка примера JSON |

| Профиль пользователя | `.dropdown` | Выпадающее меню пользователя |

## 2. Чек-лист для ручного тестирования

### 2.1. Поле ввода `#jsonInput`
- [ ] Ручной ввод текста
- [ ] Вставка через Ctrl+V / Cmd+V
- [ ] Удаление текста (Backspace, Delete, Ctrl+A + Delete)
- [ ] Отображение placeholder при пустом поле

### 2.2. Кнопка `#formatBtn` — «Отформатировать»
- [ ] Валидный JSON → форматирование с отступами
- [ ] Валидный JSON → построение дерева в `#jsonTreeContainer`
- [ ] Валидный JSON → очистка блока `#errorMsg`
- [ ] Невалидный JSON → сообщение об ошибке в `#errorMsg`
- [ ] Невалидный JSON → дерево не строится / очищается
- [ ] Пустое поле → сообщение об ошибке
- [ ] Повторное нажатие → дерево перестраивается корректно

### 2.3. Кнопка `#clearBtn` — «Очистить»
- [ ] Очищает `#jsonInput`
- [ ] Очищает `#jsonTreeContainer` (показывает плейсхолдер)
- [ ] Очищает `#errorMsg`
- [ ] Не вызывает форматирование

### 2.4. Кнопка `#exampleBtn` — «Пример»
- [ ] Вставляет предустановленный JSON в `#jsonInput`
- [ ] Заменяет существующий текст (не дописывает)
- [ ] Не форматирует автоматически
- [ ] Не строит дерево

### 2.5. Древовидная структура `#jsonTreeContainer`
- [ ] Объекты имеют сворачиваемые узлы
- [ ] Массивы отображают длину: `array[N]`
- [ ] Строки отображаются в кавычках: `"значение"`
- [ ] Числа отображаются без кавычек
- [ ] `null`, `true`, `false` отображаются корректно
- [ ] При клике на узел он сворачивается/разворачивается
- [ ] После `#clearBtn` дерево очищается

### 2.6. Кнопка `#togglePanelBtn` — «Скрыть редактор JSON»
- [ ] Скрывает левую панель (`.input-panel`)
- [ ] Текст кнопки меняется (например, на «Показать редактор JSON»)
- [ ] Повторный клик показывает панель обратно
- [ ] Состояние сохраняется после перезагрузки страницы (localStorage)

### 2.7. Профиль и хедер
- [ ] Отображается имя пользователя (из бэкенда)
- [ ] При клике на `.dropbtn` открывается выпадающее меню
- [ ] В меню есть ссылки: «Мой профиль», «На главную», «Выйти»
- [ ] Клик по «Выйти» разлогинивает пользователя

### 2.8. Обработка ошибок

| Входные данные | Ожидание |
|----------------|----------|
| `{ "a": 1` (незакрытая скобка) | Сообщение об ошибке |
| `{ "a": 1, }` (лишняя запятая) | Сообщение об ошибке |
| `{ a: 1 }` (ключ без кавычек) | Сообщение об ошибке |
| `{ 'a': 1 }` (одинарные кавычки) | Сообщение об ошибке |
| `{ "a": undefined }` (неверный тип) | Сообщение об ошибке |

### 2.9. Граничные случаи

| Входные данные | Ожидание |
|----------------|----------|
| `{}` | Пустой объект, дерево: `object {}` |
| `[]` | Пустой массив, дерево: `array[]` |
| `null` | Дерево: `null` |
| `true` / `false` | Дерево: `true` / `false` |
| `123` | Дерево: `123` |
| `"просто строка"` | Дерево: `"просто строка"` |
| Очень длинная строка (1000+ символов) | Не крашится, строка может быть обрезана |
| Глубокая вложенность (20+ уровней) | Все уровни отображаются и сворачиваются |

### 2.10. UI / UX
- [ ] Все кнопки кликабельны, имеют понятные подписи и иконки
- [ ] Фоновые элементы не мешают взаимодействию
- [ ] При длинном JSON появляется скролл
- [ ] Адаптивность: при уменьшении окна элементы не наезжают друг на друга

## 3. Тест-кейсы

| ID | Название | Предусловие | Шаги | Ожидаемый результат |
|----|----------|-------------|------|----------------------|
| **TC-01** | Валидный JSON — объект | Страница открыта | 1. Ввести `{"name":"QA","version":1}`<br>2. Нажать «Отформатировать» | Дерево содержит `name` → `"QA"` и `version` → `1`<br>Блок ошибок пуст |
| **TC-02** | Валидный JSON — массив | Страница открыта | 1. Ввести `["apple","banana","cherry"]`<br>2. Нажать «Отформатировать» | Дерево: `array[3]`<br>При развороте: `0`→`"apple"`, `1`→`"banana"`, `2`→`"cherry"` |
| **TC-03** | Невалидный JSON (незакрытая скобка) | Страница открыта | 1. Ввести `{"name":"QA"`<br>2. Нажать «Отформатировать» | В `#errorMsg` появляется сообщение об ошибке<br>Дерево не построено или очищено |
| **TC-04** | Пустое поле | Страница открыта, поле пустое | 1. Нажать «Отформатировать» | В `#errorMsg` появляется сообщение «Введите JSON» |
| **TC-05** | Кнопка «Очистить» | После TC-01 (дерево построено) | 1. Нажать «Очистить» | Поле ввода пустое<br>Дерево показывает плейсхолдер<br>Блок ошибок пуст |
| **TC-06** | Кнопка «Пример» | Страница открыта | 1. Нажать «Пример» | Поле ввода заполнено предустановленным JSON<br>Дерево не построено (пока не нажат «Отформатировать») |
| **TC-07** | Скрыть редактор | Страница открыта | 1. Нажать «Скрыть редактор JSON» | Левая панель (поле ввода и кнопки) скрыта<br>Правая панель с деревом видна |
| **TC-08** | Сохранение состояния панели | После TC-07 | 1. Перезагрузить страницу (F5) | Левая панель остаётся скрытой |
| **TC-09** | Сворачивание/разворачивание дерева | После TC-01 | 1. Свернуть корневой узел<br>2. Развернуть обратно | Дочерние узлы скрываются, затем снова показываются |
| **TC-10** | Вложенный объект | Страница открыта | 1. Ввести `{"level1":{"level2":{"level3":42}}}`<br>2. Нажать «Отформатировать» | Можно свернуть любой уровень вложенности<br>Значение `42` отображается на уровне `level3` |
| **TC-11** | Примитив `null` | Страница открыта | 1. Ввести `null`<br>2. Нажать «Отформатировать» | Дерево отображает `null` |
| **TC-12** | Примитив `true` / `false` | Страница открыта | 1. Ввести `true`<br>2. Нажать «Отформатировать» | Дерево отображает `true` |
| **TC-13** | Примитив — число | Страница открыта | 1. Ввести `123.45`<br>2. Нажать «Отформатировать» | Дерево отображает `123.45` |
| **TC-14** | Примитив — строка | Страница открыта | 1. Ввести `"Hello world"`<br>2. Нажать «Отформатировать» | Дерево отображает `"Hello world"` |
| **TC-15** | Пустой объект | Страница открыта | 1. Ввести `{}`<br>2. Нажать «Отформатировать» | Дерево отображает пустой объект (без дочерних узлов) |
| **TC-16** | Пустой массив | Страница открыта | 1. Ввести `[]`<br>2. Нажать «Отформатировать» | Дерево отображает `array[]` (пустой массив) |
